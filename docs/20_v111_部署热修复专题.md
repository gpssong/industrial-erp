## 2. 升级步骤 (DS918+)

### 2.1 拉取最新代码

```bash
cd /volume3/docker/erp-system
sudo git pull origin main
```

### 2.2 重新构建后端 (含新 jar)

```bash
cd backend
JAVA_HOME=/opt/homebrew/Cellar/openjdk@17/17.0.19/libexec/openjdk.jdk/Contents/Home \
mvn -f pom.xml -q package -DskipTests

# 推送新 jar 到 NAS 容器
cd target
base64 -i industrial-erp-1.0.4.jar | ssh gpssong@192.168.0.150 \
  'base64 -d > /tmp/ind.jar && \
   sudo /usr/local/bin/docker cp /tmp/ind.jar erp-backend:/tmp/app.jar && \
   sudo /usr/local/bin/docker exec erp-backend sh -c "rm -f /opt/app/app.jar && cp /tmp/app.jar /opt/app/app.jar"'

# 重启
ssh gpssong@192.168.0.150 \
  'sudo /usr/local/bin/docker compose -p erp-system -f /volume3/docker/erp-system/docker-compose.yml restart backend'
```

### 2.3 部署前端 (dist)

```bash
cd pc-web && npm run build

tar -cf - . | sshpass -p 'xxx' ssh gpssong@192.168.0.150 \
  'sudo /usr/local/bin/docker exec -i erp-pc-web sh -c \
   "mkdir -p /tmp/newdist && tar -xf - -C /tmp/newdist && \
    rm -rf /usr/share/nginx/html/assets/* /usr/share/nginx/html/index.html && \
    cp -a /tmp/newdist/. /usr/share/nginx/html/ && rm -rf /tmp/newdist"'
```

### 2.4 数据库迁移

依次在 MySQL 中执行:

```bash
ssh gpssong@192.168.0.150 'sudo /usr/local/bin/docker exec -i erp-mysql mysql -uroot -perp_root_pwd industrial_erp' \
  < sql/11_add_template_config.sql

ssh gpssong@192.168.0.150 'sudo /usr/local/bin/docker exec -i erp-mysql mysql -uroot -perp_root_pwd industrial_erp' \
  < sql/12_add_gram_weight.sql
```
