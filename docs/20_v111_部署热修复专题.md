## 8. v1.1.3 安全与事务修复 (2026-07-06)

### 背景

v1.1.2 之后的全项目静态扫描发现一批接口鉴权缺失和数据一致性问题。其中 4 个属于 CVE 级别(任意登录用户可访问敏感接口),2 个事务缺失会导致数据半成品,3 个 NPE 风险,以及若干前端用户体验问题。

### 接口鉴权补全 (CVE 级别)

| 接口 | 改动 | 文件 | 影响 |
|---|---|---|---|
| `/api/print/**` | 移除 SaToken 白名单 + 类级 `@SaCheckPermission("print:use")` | `config/SaTokenConfig.java`, `modules/print/controller/PrintController.java` | 未登录/无权限访问打印页直接 401/403 |
| `/api/report/**` | 类级 `@SaCheckPermission("report:view")` | `modules/report/controller/ReportController.java` | dashboard / salesSummary / inventorySummary 等经营数据需授权 |
| `/api/system/upload/file` | 类级 `@SaCheckLogin` + 后缀白名单移除 `.jsp`/`.html`/`.htm`/`.svg` | `modules/system/controller/SysUploadController.java` | 防 XSS 载荷 + 未鉴权上传 |
| `/api/system/backup/manual` `restore` `delete` | `@SaCheckRole("admin")` | `modules/system/controller/SysBackupController.java` | 仅 admin 可手动备份/恢复/删除 |
| `/api/system/backup/page` `list` | `@SaCheckLogin` | 同上 | 列表登录即可查 |

### 事务补全

| 文件 | 方法 | 改动 |
|---|---|---|
| `modules/purchase/service/PurReceiptService.java` | `check()` | 加 `@Transactional(rollbackFor = Exception.class)` (4 写无事务, 审核中途挂会库存已增但订单未收) |
| `modules/finance/controller/FinArapController.java` | `cash()` | 加 `@Transactional(rollbackFor = Exception.class)` (收付款 + 现金流水须同成功同失败) |

### NPE 防御 + 日志

| 文件 | 改动 |
|---|---|
| `modules/production/service/PrdOrderService.java:279` | `wh.getWarehouseName()` 加 null 保护 (成品入库仓库被删不会崩) |
| `modules/outsource/service/OutsourceService.java:92, 143` | 同上, 外协发料/入库两处 |
| `modules/system/service/AuthService.java:90` | `Integer.parseInt(failCount)` 包 try-catch, Redis 注入非数字不会阻塞登录 |
| `modules/print/service/PrintDataLoader.java` | `catch (Exception ignore) {}` 改 `log.warn(..., e)`, 数据缺失不再静默 |

### 权限 seed

新文件 `sql/seed/14_v113_permissions.sql`:

```sql
-- 1) 新增两条 F=按钮 类型菜单, perms 字段承载权限码
INSERT IGNORE INTO sys_menu (id, parent_id, menu_name, menu_type, perms, is_visible) VALUES
  (950, 0, '打印使用', 'F', 'print:use', 0),
  (951, 0, '报表查看', 'F', 'report:view', 0);

-- 2) 给所有内置业务角色 (SUPER_ADMIN/PURCHASE_MGR/SALES_MGR/WAREHOUSE_MGR/PRODUCTION_MGR/FINANCE) 自动绑定
INSERT IGNORE INTO sys_role_menu (role_id, menu_id)
SELECT r.id, m.id FROM sys_role r CROSS JOIN sys_menu m
WHERE m.perms IN ('print:use','report:view') AND r.role_code IN (...);
```

**升级步骤**:
```bash
docker exec -i erp-mysql mysql -uroot -perp_root_pwd industrial_erp < sql/seed/14_v113_permissions.sql
```

**注意**:`SUPER_ADMIN` 必须勾,否则现有 admin 用户会失去打印/报表权限(rollback)。
