# 飞牛 OS - 仅部署后端指南

> **适用场景**: 前端在你本地电脑 (或单独部署), 只需要把 **Spring Boot 后端 + MySQL + Redis** 部署到飞牛 OS.

---

## 一、为什么只部署后端

| 场景 | 适用 |
|---|---|
| 本地开发前端, 后端放飞牛供多端调用 | ✅ 你的场景 |
| 前端用 Vite 开发服务器, 跨网段联调 | ✅ 推荐 |
| 前端单独部署到 CDN / 其他服务器 | ✅ |
| 前后端都要部署到飞牛 | 看 `docs/18_飞牛OS部署指南.md` |

---

## 二、架构

```
┌─────────────────────────────────────┐
│  飞牛 OS (NAS)                      │
│                                     │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐
│  │  backend │  │  mysql   │  │  redis   │
│  │  :8080   │←→│  :3306   │  │  :6379   │
│  │ Spring   │  │ (内部)   │  │ (内部)   │
│  │ Boot     │  └──────────┘  └──────────┘
│  └────┬─────┘
│       │
│       :8080 (对外暴露, 给前端调用)
└───────┼─────────────────────────────┘
        │ HTTP (局域网 / 公网)
        ▼
┌──────────────────┐
│  前端 (你电脑)   │
│  Vite / Nginx    │
│  浏览器访问      │
└──────────────────┘
```

> MySQL/Redis 端口 **不暴露主机**, 仅容器内网互通, 安全.

## 三、上传项目到飞牛

只传必要目录:

```
erp-system/                    ← 项目根目录 (你传这个到飞牛)
├── docker-compose.backend.yml ← 仅后端 compose 文件 (新增)
├── .env.example               ← 环境变量样例
├── backend/                   ← Spring Boot 源码 + Dockerfile
│   ├── Dockerfile
│   ├── pom.xml
│   └── src/
└── sql/                       ← 数据库初始化脚本
```

### 方式 A: SSH + scp (推荐, 一次性)

在你 **macOS 终端** (不是飞牛 SSH 里):

```bash
cd /Users/tongban/Documents/根据前端开发erp 2

# 上传整个项目 (首次, 几十 MB, 看带宽约 1~5 分钟)
scp -r erp-system admin@NAS_IP:/vol1/docker/

# 如果 admin 用户没 /vol1/docker 写权限, 先 ssh 进去 mkdir
ssh admin@NAS_IP "sudo mkdir -p /vol1/docker && sudo chown admin /vol1/docker"
```

### 方式 B: git clone (推荐, 升级方便)

```bash
# macOS: 先把代码推到 git 仓库 (GitHub/Gitee/公司 GitLab)
cd /Users/tongban/Documents/根据前端开发erp 2
git remote add origin https://你的仓库地址.git   # 第一次需要
git push -u origin main

# 飞牛 SSH 里
ssh admin@NAS_IP
sudo -i
cd /vol1/docker
git clone https://你的仓库地址.git erp-system
cd erp-system
```

### 方式 C: SMB 上传 (最直观)

1. 飞牛桌面 → **设置** → **SMB** → 启用
2. macOS Finder → 前往 → 连接服务器 → `smb://NAS_IP/docker`
3. 把 `erp-system` 文件夹拖进去

## 四、配置 .env

```bash
ssh admin@NAS_IP
sudo -i
cd /vol1/docker/erp-system
cp .env.example .env
vi .env      # 改 MYSQL_ROOT_PASSWORD / SPRING_DATASOURCE_PASSWORD 为强密码
```

## 五、一键部署

```bash
cd /vol1/docker/erp-system
./scripts/fnos-backend-deploy.sh
```

脚本会自动:
1. ✅ 检查 docker / compose 环境
2. ✅ 校验项目文件齐全
3. ✅ 确认 .env 已配置
4. ✅ `docker compose -f docker-compose.backend.yml up -d --build`
5. ✅ 等待 30 秒后展示 3 个容器状态

首次构建 5~10 分钟 (N100 / i5 级别), 主要是下载 Maven 依赖.

## 七、前端对接 (关键)

### 7.1 本地 Vite 开发 (你电脑)

编辑 `pc-web/.env.development`:

```ini
# 后端 API 地址 (换成你的飞牛 IP)
VITE_API_BASE_URL=http://192.168.1.100:8080/api
```

CORS: 后端 `application.yml` 默认已允许 `localhost:5173` / `localhost:8080`, 本地开发无需额外配置.

### 7.2 局域网多端访问 (手机/平板/PDA)

只要连同一个 Wi-Fi, 前端配置:

```ini
VITE_API_BASE_URL=http://192.168.1.100:8080/api
```

### 7.3 公网 HTTPS 部署 (可选)

飞牛 → **设置** → **反向代理**:

| 字段 | 值 |
|---|---|
| 来源 | HTTPS, 443, 主机名 `api.你的域名.com` |
| 目的地 | HTTP, 127.0.0.1, **8080** |

然后前端配置:

```ini
VITE_API_BASE_URL=https://api.你的域名.com/api
```

CORS: 在 `.env` 里追加 (Spring 不支持通配符):

```ini
ERP_CORS_ALLOWED_ORIGINS=https://你的前端域名.com
```

## 八、常见问题

### 8.1 端口 8080 已被占用
修改 `docker-compose.backend.yml`:
```yaml
ports:
  - "8888:8080"   # 主机 8888 映射到容器 8080
```
前端对应改成 `http://NAS_IP:8888/api`.

### 8.2 前端跨域报错 (CORS)
浏览器 F12 → Console → 看 `Access-Control-Allow-Origin` 报错.

在飞牛上编辑 `erp-system/.env`:
```ini
ERP_CORS_ALLOWED_ORIGINS=http://localhost:5173,http://192.168.1.100:8080
```
> 注意: 完整 Origin (含协议), 逗号分隔, 不要空格.

重启:
```bash
docker compose -f docker-compose.backend.yml restart backend
```

### 8.3 后端连接 MySQL 失败 (Communications link failure)
等 30~60 秒, MySQL 健康检查未通过, 后端会自动重试. 如持续报错:
```bash
docker logs erp-mysql | tail -50   # 看 MySQL 启动日志
docker logs erp-backend | tail -50 # 看后端连接日志
```
常见原因:
- `.env` 里 `MYSQL_ROOT_PASSWORD` 和 `SPRING_DATASOURCE_PASSWORD` 不一致
- 内存不足, MySQL 被 OOM killed

### 8.4 内存不足 (8G 机型常见)
编辑 `docker-compose.backend.yml`:
```yaml
mysql.command:
  - --innodb-buffer-pool-size=384M    # 原 512M 改 384M
backend.environment.JAVA_OPTS: "-Xms256m -Xmx512m -XX:MaxMetaspaceSize=160m ..."
```

### 8.5 想清空数据重来
```bash
cd /vol1/docker/erp-system
docker compose -f docker-compose.backend.yml down -v   # -v 删数据卷
rm -rf data/
docker compose -f docker-compose.backend.yml up -d --build
```

### 8.6 日志查看技巧
```bash
# 后端最近 100 行
docker logs --tail 100 erp-backend

# 实时跟踪 (Ctrl+C 退出, 容器继续跑)
docker logs -f erp-backend

# MySQL 慢查询
docker exec erp-mysql mysql -uroot -p密码 -e "SHOW VARIABLES LIKE 'slow_query%';"
```

## 九、升级流程

```bash
cd /vol1/docker/erp-system

# 1. 数据库备份
docker exec erp-mysql mysqldump -uroot -p你的密码 industrial_erp > db_backup_$(date +%Y%m%d).sql

# 2. 拉新代码 (git 方式)
git pull origin main

# 3. 滚动重启
docker compose -f docker-compose.backend.yml up -d --build

# 4. SQL 变更 (如有)
docker exec -i erp-mysql mysql -uroot -p你的密码 industrial_erp < sql/新文件.sql
```

## 十、与全栈部署的差异

| 项目 | 全栈部署 (`docker-compose.yml`) | 后端部署 (`docker-compose.backend.yml`) |
|---|---|---|
| 服务数 | 4 个 (mysql + redis + backend + pc-web) | 3 个 (mysql + redis + backend) |
| 端口暴露 | 80 (前端) + 8080 (后端) | 仅 8080 (后端) |
| 前端位置 | 飞牛上 nginx 托管 | 你本地电脑 |
| 适合 | 正式生产环境 | 开发联调 / 前后端分离部署 |
| CORS 配置 | 简单 (同域) | 需要配 (跨域) |
