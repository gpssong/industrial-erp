## 二、Mac 环境搭建

### 2.1 使用 Homebrew 一键安装 (推荐)
```bash
# 安装 Homebrew (如未安装)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 安装必备工具
brew install openjdk@17 maven node@20 pnpm mysql redis git

# 链接 java
sudo ln -sfn $(brew --prefix openjdk@17)/libexec/openjdk.jdk /Library/Java/JavaVirtualMachines/openjdk-17.jdk
export JAVA_HOME=$(brew --prefix openjdk@17)/libexec/openjdk.jdk
echo 'export JAVA_HOME=$(brew --prefix openjdk@17)/libexec/openjdk.jdk' >> ~/.zshrc
source ~/.zshrc

# 启动 MySQL/Redis
brew services start mysql
brew services start redis

# 配置 MySQL
mysql -uroot -e "ALTER USER 'root'@'localhost' IDENTIFIED BY 'root'; FLUSH PRIVILEGES;"
```

### 2.2 使用 Docker (推荐)
```bash
# 安装 Docker Desktop for Mac
# 官网下载: https://www.docker.com/products/docker-desktop

# 启动 MySQL + Redis
docker run -d --name erp-mysql \
  -e MYSQL_ROOT_PASSWORD=root \
  -e MYSQL_DATABASE=industrial_erp \
  -p 3306:3306 \
  mysql:8.0 --character-set-server=utf8mb4 --collation-server=utf8mb4_unicode_ci

docker run -d --name erp-redis \
  -p 6379:6379 \
  redis:7-alpine

# 验证
docker ps
mysql -h127.0.0.1 -uroot -proot -e "SELECT VERSION();"
redis-cli ping
```

### 2.3 拉取代码
```bash
cd ~/Documents
# git clone <repo-url> industrial-erp
cd industrial-erp/erp-system
```

### 2.4 初始化数据库
```bash
# 方式 A: 本机 MySQL, 依次执行
for f in sql/0*.sql; do
  echo "导入 $f"
  mysql -uroot -p industrial_erp < "$f"
done

# 方式 B: 一条命令
cat sql/0*.sql | mysql -uroot -p industrial_erp

# 方式 C: Docker
for f in sql/0*.sql; do
  docker exec -i erp-mysql mysql -uroot -proot industrial_erp < "$f"
done
```

### 2.5 启动后端
```bash
cd backend

# 创建 application-local.yml (本机配置)
cat > src/main/resources/application-local.yml << 'EOF'
spring:
  datasource:
    url: jdbc:mysql://127.0.0.1:3306/industrial_erp?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
    username: root
    password: root
  data:
    redis:
      host: 127.0.0.1
      port: 6379
EOF

# 用 IDE (IntelliJ IDEA) 打开, 找到 IndustrialErpApplication.java 右键 Run
# 或命令行
mvn spring-boot:run
```

启动成功后:
- API 文档: http://localhost:8080/api/doc.html
- 健康检查: http://localhost:8080/api/auth/captcha

## 三、Linux (Ubuntu 22.04) 环境搭建

```bash
sudo apt update
sudo apt install -y openjdk-17-jdk maven nodejs npm git mysql-server redis-server

# nvm 管理 Node
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
source ~/.bashrc
nvm install 20
nvm use 20
npm install -g pnpm

# MySQL 配置
sudo mysql_secure_installation
sudo mysql -uroot -p
> ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY 'root';
> FLUSH PRIVILEGES;

# 启动
sudo systemctl start mysql
sudo systemctl start redis
sudo systemctl enable mysql redis

# 验证
java -version
mvn -v
node -v
```

后续步骤同 Mac。

## 四、Windows 11 环境搭建

## 七、常见启动问题

### 7.1 Communications link failure (MySQL)
- 检查 MySQL 是否启动
- 检查端口 3306 是否通
- 验证账号密码

### 7.2 Redis connection refused
```bash
redis-server --daemonize yes
redis-cli ping
```

### 7.3 EACCES permission denied (Mac/Linux)
```bash
sudo chown -R $USER:staff ~/.npm
sudo chown -R $USER:staff ~/.config
```

### 7.4 端口被占用
```bash
lsof -i :5173
# 修改 vite.config.js 的 server.port
```

### 7.5 登录提示用户名或密码错误
- 确认 09_seed_data.sql 已执行
- 确认数据库字符集是 utf8mb4
- 重新执行: mysql -uroot -p industrial_erp < sql/09_seed_data.sql

### 7.6 接口 401 Unauthorized
- Token 过期, 重新登录
- 前端 axios baseURL 应为 /api
- 检查后端 Sa-Token exclude 路径

### 7.7 跨域 CORS
- 确认 SaTokenConfig.addCorsMappings 已配置
- 确认前端走 Vite 代理

## 十、生产环境快速检查清单

部署到生产前:
- 修改 application.yml 的数据库密码
- 修改 sa-token.jwt-secret-key 为强随机字符串
- 关闭调试日志
- 配置 Nginx HTTPS
- 启用 MySQL 主从
- 启用 Redis 持久化
- 配置日志收集
- 配置监控告警
- 配置备份策略
- 防火墙只开放 80/443/22
