## 一、自服务修改密码

### 入口

`pc-web` 右上角 → 点头像 → 下拉菜单:
```
┌──────────────────┐
│ 🔑 修改密码       │
├──────────────────┤
│ ⏏ 退出登录       │
└──────────────────┘
```

点"修改密码"弹窗输入 **原密码** + **新密码** + **确认新密码**,提交后强制重新登录。

### 后端接口

| 项 | 值 |
|---|---|
| Method | `PUT` |
| URL | `/api/system/user/me/password` |
| Auth | 必须登录 (Sa-Token) |
| Body | `{ "oldPassword": "...", "newPassword": "..." }` |
| 成功 | 200 操作成功 |
| 失败 500 | 原密码错误 / 新密码不能与原密码相同 / 新密码至少 6 位 |
| 实现 | `backend/src/main/java/com/industrial/erp/modules/system/controller/SysUserController.java` `@PutMapping("/me/password")` |
| Service | `SysUserService.changeOwnPassword(oldPwd, newPwd)` |

### 安全设计

- **校验旧密码**: BCrypt 匹配 `user.password`,错误时抛 BizException("原密码错误")
- **不需权限**: 用户改自己密码不需要任何 `system:user:*` perm
- **强密码要求**: 前端校验 ≥6 位;后端额外校验"新密码不能与原密码相同"
- **改后强制登出**: 前端提交成功后清空 token,跳到 /login — 防止 token 仍在用的旧会话持有者继续访问

### 与现有"管理员改用户密码"的区别

| 接口 | 谁用 | 权限 |
|---|---|---|
| `PUT /system/user/{id}/password` | 管理员改他人密码 | `system:user:edit` |
| `PUT /system/user/me/password` | 用户改自己密码 | 只需登录 |
| `POST /system/user/{id}/resetPwd?newPwd=...` | 超管重置他人(默认 123456) | `requireSuperAdmin()` |
| `POST /auth/setpwd` | 超管通过登录 API 改(临时接口,白名单已移除) | 需登录 + 超管 |

### 前端改动

| 文件 | 改动 |
|---|---|
| `pc-web/src/layouts/MainLayout.vue` | dropdown 加 `<el-dropdown-item command="changePassword">`, + `<el-dialog>` 修改密码弹窗, + onCommand 处理 + submitChangePassword 函数 |
| `pc-web/src/api/system.js` | `userApi.changeOwnPassword(oldPassword, newPassword)` |

## 二、静默 ResizeObserver 警告

### 现象

浏览器 console 频繁出现:
```
[Error] ResizeObserver loop completed with undelivered notifications. (home.93gushi.com, line 0, x5)
```

### 原因

Element Plus 的 `el-tabs` / `el-table` / `el-dialog` 在 mount/重排时会触发 ResizeObserver。
当一帧内通知多次产生且 callback 还没执行完,浏览器会发出这个原生 warning。
**无害,只是噪声**。

### 修复

`pc-web/src/main.js` 全局过滤:
```js
const _origError = window.console.error
window.console.error = (...args) => {
  const msg = args[0]
  if (typeof msg === 'string' && msg.includes('ResizeObserver loop')) return
  _origError.apply(window.console, args)
}
window.addEventListener('error', (e) => {
  if (e && e.message && e.message.includes('ResizeObserver loop')) {
    e.stopImmediatePropagation()
    e.preventDefault()
  }
}, true)
```

### 为什么不在源头修

也可以从源头解决,比如:
- 给 `el-table` 加 `key` 强制重建
- 用 `ResizeObserver` polyfill (e.g. `resize-observer-polyfill`)
- 给 `el-dialog` 加 `append-to-body`

这些都不通用 — 加 console 过滤是最稳的"治标不治本"方案,不影响功能,后续 Element Plus 升级也无需修改。

## 部署清单

### 1. 后端 jar 重新部署

无 SQL 变更,只需重新打 jar:
```bash
# mac 本地
cd backend && mvn -DskipTests clean package

# 推 NAS
# (1) mac 起 HTTP 服务: cd backend/target && python3 -m http.server 9879
# (2) NAS wget: cd /volume3/docker/erp-system/backend \
#     && wget -qO industrial-erp-1.0.4.jar http://MAC_IP:9879/industrial-erp-1.0.4.jar

# 重 build + recreate
cd /volume3/docker/erp-system
sudo docker compose -p erp-system build --no-cache backend
sudo docker compose -p erp-system up -d --force-recreate --no-deps backend
```

### 2. 前端 dist 重新部署

```bash
# mac 本地
cd pc-web && npm run build
cd dist && tar czf /tmp/pc-web-dist.tgz .
python3 -m http.server 9879 --directory /tmp --bind 0.0.0.0

# NAS 拉 + 解压
cd /volume3/docker/erp-system/pc-web
mv dist dist.old && mkdir dist.new && cd dist.new \
  && wget -qO- http://MAC_IP:9879/pc-web-dist.tgz | tar xzf -
cd /volume3/docker/erp-system/pc-web
mv dist dist.new && mv dist.new dist   # 实际是 dist.new -> dist
# (swap 顺序: mv dist dist.old && mv dist.new dist)

# 重 build + recreate
cd /volume3/docker/erp-system
sudo docker compose -p erp-system build --no-cache pc-web
sudo docker compose -p erp-system up -d --force-recreate --no-deps pc-web
```
