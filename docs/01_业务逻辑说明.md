# 关键业务逻辑说明

## 1. 库存核心 - 防超卖

### 1.1 双重防护
1. **Redis 分布式锁** (RedisLock):
   - Key: `erp:stock:lock:{warehouseId}:{productId}:{batchNo}`
   - 防止同一商品并发请求导致的脏读
2. **MySQL 行锁** (SELECT ... FOR UPDATE):
   - `inv_stock` 按 (warehouseId, locationId, productId, batchNo) 唯一索引
   - SELECT...FOR UPDATE 加悲观锁, 阻塞其他出库事务

### 1.2 业务流程 (出库为例)
```
SalDeliveryService.check(id)
  ├─ 1. 校验客户信用额度
  ├─ 2. 预占信用 (incrCreditUsed)
  ├─ 3. 循环明细: StockService.outStock()
  │    └─ RedisLock (5s 等待 / 30s 持有)
  │         ├─ SELECT...FOR UPDATE 锁库存行
  │         ├─ 校验: stock.qty >= qty  (禁止负库存)
  │         ├─ UPDATE inv_stock SET qty -= ?, totalCost -= avgCost * qty
  │         ├─ INSERT inv_ledger  (流水)
  │         └─ UPDATE base_product SET costPrice = ? (平均成本)
  ├─ 4. 计算毛利: profit = totalAmount - totalCost
  └─ 5. FinArapService.createArForSales()  (应收台账)
```

### 1.3 移动加权平均成本 (Moving Weighted Average)
```
入库时:
  new_avg_cost = (current_total_cost + new_amount) / (current_qty + new_qty)
  
  特殊处理:
    - current_qty = 0: 直接使用本次单价
    - 退货入库 (方向-1): 按当前出库价冲减
    
出库时:
  out_cost = avg_cost * qty
  成本金额不参与平均成本计算 (出库不重算均价)
```

## 2. 信用额度控制
- 客户表保存 `creditLimit` (授信额度) / `creditUsed` (已用)
- 销售出库时实时校验: `creditUsed + thisBill > creditLimit` 则禁止开单
- 可通过 `sys.credit.overdraft` 系统配置切换严格/宽松模式
- 审核通过后 `creditUsed += totalAmountTax`; 收款核销时扣减

## 3. 多单位换算
- 商品表保存 `mainUnitId` (主单位)
- `base_product_unit` 保存多单位换算: `1 主单位 = conversionRate 此单位`
- 例: BOPP薄膜 1卷 = 4000米, 销售时按"米"开单, 库存按"卷"汇总
- 公式: `qty_main = qty_unit / conversionRate`

## 4. 工业行业特性
### 4.1 米重换算 (薄膜行业)
- 公式: `重量(kg) = 长度(m) × 幅宽(m) × 厚度(mm) × 密度(g/cm³) / 1000`
- 应用: 采购按"吨"入库, 销售按"卷/米"出库, 库存按主单位汇总

### 4.2 分切/复卷/裁切
- 单据类型: `inv_cut_process` + `inv_cut_process_detail`
- 一行原料消耗 (direction=-1), 一行成品产出 (direction=+1)
- 自动计算: `loss_qty = in_qty - out_qty`, `loss_rate = loss / in_qty * 100%`

### 4.3 批次 + 有效期
- 库存表保留 `batchNo / productionDate / expireDate`
- 定时任务扫描 `expire_date < NOW() + 30天` -> 写入 `inv_warning` -> 推送通知

## 5. 财务往来 (应收/应付)
### 5.1 自动生成
- 采购入库审核 -> AP 应付
- 销售出库审核 -> AR 应收

### 5.2 收/付款核销
- `fin_cash_flow` (收/付款单) -> `fin_cash_writeoff` (核销明细) -> `fin_arap` (应收/应付更新)
- 一笔收/付款可核销多张单据; 一张单据可被多次核销
- 核销完成自动更新 `bill_status`: UNPAID -> PARTIAL -> PAID

## 6. 生产管理 - BOM 展开
```
prd_order.release(orderId)
  ├─ 读 BOM 明细
  ├─ 计算 ratio = planQty / bom.baseQty
  ├─ 实际用量 = baseQty * ratio * (1 + lossRate/100)
  ├─ 自动生成 prd_requisition (领料单) 多行
  └─ 状态: DRAFT -> RELEASED

prd_order.finish(orderId, goodQty, lossQty)
  ├─ 生成 prd_finished_in (成品入库单)
  ├─ 价格 = 归集成本 / 良品数
  ├─ StockService.inStock()  (成品入库)
  └─ 状态 -> FINISHED
```

## 7. 并发与性能
- 库存查询走主键/唯一索引
- 分页查询: MyBatis-Plus 分页插件, 单页最多 500
- 字典/配置: Redis 缓存 (启动时加载)
- 报表查询: 走汇总表 + 索引, 大数据量可走 ClickHouse 扩展

## 8. 审计与软删除
- 所有表 `deleted TINYINT(0/1)`, MP 全局逻辑删除
- 唯一索引统一包含 `deleted` 字段
- 审计字段: `create_by / create_time / update_by / update_time`
- 操作日志: AOP 切面 + `sys_oper_log` (可由 `erp.log.oper=true` 启用)

## 9. 多租户预留
- 所有表 `tenant_id BIGINT`
- MP 多租户插件 (待启用): `MybatisPlusInterceptor.addInnerInterceptor(new TenantLineInnerInterceptor(...))`
- 数据隔离: 默认按 `tenant_id = 1` (单租户), 后续可升级 SaaS
