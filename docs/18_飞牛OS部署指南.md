# 飞牛 OS (fnOS) 部署指南

> 飞牛 OS 是 2024 年推出的国产 NAS 系统, 基于 Debian Linux + btrfs, 原生支持 Docker Compose. **直接兼容本项目 docker-compose.yml**, 部署流程比群晖还简单, 不存在 Container Manager 的镜像标签兼容性问题.

---

## 一、飞牛 OS 简介

| 特性 | 说明 |
|---|---|
| 厂商 | 北京飞牛天下科技 (fnOS.cn) |
| 内核 | Debian 12 (Bookworm) |
| 文件系统 | btrfs (默认) / ext4 |
| Docker | 原生 Compose V2, 无版本兼容问题 |
| 价格 | 完全免费 (无群晖那种 DSM 授权费) |
| 客户端 | Web 桌面 (桌面端)、fnOS App (移动端) |
| 适用机型 | x86_64 台式机/小主机/老 NAS, 资源占用比群晖低 |

## 二、硬件准备

### 2.1 推荐配置 (与群晖 DS918+ 对标)

| 部件 | 最低 | 推荐 | 备注 |
|---|---|---|---|
| CPU | x86_64 双核 | Intel N100 / N305 | 本项目按 J3455 调优, N100 性能翻倍 |
| 内存 | 8G DDR4 | **16G** | MySQL(1G) + Redis(0.3G) + 后端(1.2G) + 系统(2G) + 前端(0.1G) ≈ 5G |
| 系统盘 | 64G SSD | 128G NVMe | 装系统 + Docker 镜像缓存 |
| 数据盘 | 1 块 HDD | 2~4 块 HDD (RAID1/5/10) | 业务数据, btrfs 支持软 RAID |
| 网络 | 千兆网口 | 千兆 + 2.5G | 视并发量, 单机用千兆够用 |
| 启动 U 盘 | USB 2.0+ | USB 3.0 | 装系统用, 装完可拔 |

### 2.2 适用场景
- ✅ **闲置台式机 / 小主机** (NUC / 畅网 / 倍控 / 暴风等) 最划算
- ✅ **老 NAS** (群晖、威联通退役机也能刷飞牛)
- ✅ **普通 PC + UPS** 也能当 NAS 用
- ❌ 不适合 ARM 设备 (本项目 Java 后端 + 多阶段构建, 强烈建议 x86_64)

### 2.3 推荐机型参考

| 机型 | 价格区间 | 说明 |
|---|---|---|
| 倍控 N100 准系统 | ¥600~800 | 4 网口, 自带 N100, 性价比之选 |
| 畅网 N305 | ¥800~1200 | 多盘位, 适合 4 盘 NAS |
| 暴风 N95/N100 | ¥500~700 | 单盘/双盘位, 适合家用 |
| 退役台式机 i5-6500 | ¥300~500 | 二手价格低, 加块硬盘就够 |

## 三、飞牛 OS 安装

### 3.1 下载镜像

1. 官网: https://www.fnos.cn
2. 下载页 → **fnOS 0.8.x 正式版** (推荐最新稳定版, 不要用 Beta)
3. 文件名形如 `fnos-0.8.6-x86_64.iso` 或类似

### 3.2 制作启动 U 盘

**macOS:**
```bash
# 下载 balenaEtcher: https://etcher.balena.io
# 打开 → 选 ISO → 选 U 盘 → Flash!
```

**Windows:**
- 用 Rufus 或 balenaEtcher 把 ISO 写到 U 盘

**Linux:**
```bash
sudo dd if=fnos-0.8.6-x86_64.iso of=/dev/sdX bs=4M status=progress
sync
```

### 3.3 BIOS 设置

开机按 Del/F2 进 BIOS:

| 项 | 设置 |
|---|---|
| Boot Mode | UEFI (推荐) 或 Legacy 都行 |
| Secure Boot | **Disabled** (必关) |
| SATA Mode | AHCI |
| 虚拟化 (VT-x / AMD-V) | Enabled |
| 启动顺序 | USB 第一 |

### 3.4 安装流程

1. U 盘启动 → 看到 fnOS 安装菜单 → **Install fnOS** (图形界面)
2. **目标盘选择**:
   - ⚠️ **务必选对盘!** 数据会被清空
   - 推荐用单独一块 SSD 装系统, 不要和数据盘混用
3. **分区方案**: 默认 (btrfs, 自动 subvolume)
4. 等待 3~5 分钟安装完成
5. 拔掉 U 盘 → 重启

### 3.5 首次配置

1. 重启后, 同一局域网 PC 浏览器访问 `http://fnos.local` 或 `http://NAS_IP`
2. 设置管理员账号密码 (强密码!)
3. 进入 **桌面** → 看到类似群晖 DSM 的图标菜单

## 四、初始化存储池 (数据盘)

如果用了独立的 HDD 数据盘, 需要创建存储池:

1. **桌面** → **文件管理** → 左侧 **存储空间**
2. 新建存储池 → 选数据盘 → RAID 类型:
   - 单盘: Basic (无冗余, 容量最大)
   - 两盘: Mirror (RAID1, 冗余)
   - 三盘以上: RAID5 / RAID10 (推荐)
3. 新建存储空间 → 挂载到 `/vol1` 或 `/vol2`

> 安装项目前先创建存储池, 项目文件和数据都放这里, 不要放系统盘.

## 五、上传项目文件

### 方式 A: 通过 SMB (推荐, 最简单)

#### 在 fnOS 端开启 SMB

1. **桌面** → **设置** → **SMB** → 启用 → 设置共享名 `docker`
2. macOS: Finder → 前往 → 连接服务器 → `smb://fnos.local/docker` 或 `smb://NAS_IP/docker`
3. Windows: 文件夹地址栏输入 `\\fnos.local\docker`

#### 在本机 clone / 上传

```bash
# macOS 终端, 假设挂载到 /Volumes/docker
cd /Volumes/docker
git clone https://你的仓库地址/erp-system.git
cd erp-system
cp .env.example .env
# 用文本编辑器改密码
nano .env   # 或 vim .env
```

### 方式 B: SSH (推荐, 不受网络环境限制)

#### 在 fnOS 开启 SSH

1. **桌面** → **设置** → **SSH** → 启用 → 端口 22
2. (可选) **设置** → **终端** → 启用 fnOS 终端

#### 在本机推送

```bash
# macOS / Linux
scp -r /本地路径/erp-system admin@NAS_IP:/vol1/docker/

# 或者 git clone
ssh admin@NAS_IP
sudo -i
cd /vol1/docker
git clone https://你的仓库地址/erp-system.git
cd erp-system
cp .env.example .env
vi .env       # 改密码
```

> ⚠️ **路径说明**:
> - btrfs 存储池挂载点是 `/vol1`、`/vol2` (类似群晖的 `/volume1`)
> - 系统盘是 `/` 根分区, **不要放项目**, 系统盘小且不可 RAID 备份
> - 容器数据用 `./data/` bind mount, 就在项目目录里, 自然落在数据盘上

## 六、配置 .env

### 6.1 必改项

```ini
# 强密码 (字母+数字+符号, 16 位以上)
MYSQL_ROOT_PASSWORD=YourStr0ng!P@ssw0rd

# 必须与上面一致
SPRING_DATASOURCE_PASSWORD=YourStr0ng!P@ssw0rd
```

### 6.2 公网访问时改 CORS (可选)

```ini
ERP_CORS_ALLOWED_ORIGINS=https://erp.你的域名.com
```

> Spring CORS 因 `allowCredentials=true` 不支持通配符, 必须逐个列出完整 Origin (含协议).

## 七、启动项目

### 方式 A: 桌面图标 (推荐新手)

1. **桌面** → **Docker** → **Compose** → **创建项目**
2. 路径选 `/vol1/docker/erp-system`
3. 勾选 `docker-compose.yml` (fnOS 自动识别 V2 格式)
4. 点击 **构建并启动** → 等待 5~10 分钟首次构建
5. 在 **容器** 页面查看 4 个容器是否都 healthy/running

### 方式 B: SSH + docker compose (推荐熟手)

```bash
ssh admin@NAS_IP
sudo -i
cd /vol1/docker/erp-system

# 检查 docker compose 版本 (fnOS 自带 V2)
docker compose version
# Docker Compose version v2.x.x ...

# 构建并后台启动
docker compose up -d --build

# 查看运行状态
docker compose ps

# 实时查看后端日志 (Ctrl+C 退出, 容器继续跑)
docker compose logs -f backend
```

### 首次构建时间参考

| 机型 | 首次构建 | 二次构建 (有缓存) |
|---|---|---|
| N100 / N305 | 3~5 分钟 | 30 秒~1 分钟 |
| i5-6500 | 5~8 分钟 | 1~2 分钟 |
| J3455 (DS918+ 同款) | 8~15 分钟 | 2~3 分钟 |

> 首次构建慢是因为要下载 maven 依赖 (~500MB) 和 npm 依赖 (~200MB). 飞牛比群晖好的地方: **下载速度不受限**, 镜像源默认走 docker.io, 国内可达.

## 九、外网访问 (HTTPS)

### 9.1 反向代理

**桌面** → **设置** → **反向代理** → 新建:

| 字段 | 值 |
|---|---|
| 来源 | HTTPS, 443, 主机名 `erp.你的域名.com` |
| 目的地 | HTTP, 127.0.0.1, 80 |

### 9.2 证书 (Let's Encrypt 免费申请)

**设置** → **证书** → 申请 → 填域名 → 自动签发 → 绑定到反代

### 9.3 防火墙

**设置** → **防火墙** → 放行 443, **不放行** 80 / 8080 / 3306 / 6379.

### 9.4 DDNS (动态域名解析)

**设置** → **DDNS** → 添加:
- 阿里云 / 腾讯云 / Cloudflare 都支持
- 域名解析到 NAS 公网 IP
- 路由器开端口转发 443 → NAS_IP

## 十、备份策略 (三层都做)

### 10.1 系统层: btrfs 快照 (推荐)

```bash
# 手动快照
sudo btrfs subvolume snapshot -r /vol1/docker/erp-system /vol1/docker/.snapshots/erp-$(date +%Y%m%d)
```

升级前打一个快照, 出问题 30 秒回滚.

### 10.2 应用层: ERP 自带备份

登录 ERP → 系统设置 → 数据备份 → 每天 1 次, 保留 30 天. 落在 `/vol1/docker/erp-system/data/backup/`.

### 10.3 异地备份: 手动 / rsync / 飞牛自带

**桌面** → **备份** → 选择目标 (外接 USB / 另一台飞牛 / S3 云存储).

## 十一、升级流程

```bash
ssh admin@NAS_IP
sudo -i
cd /vol1/docker/erp-system

# 0. 打快照 (强烈推荐, 1 秒回滚)
sudo btrfs subvolume snapshot -r /vol1/docker/erp-system /vol1/docker/.snapshots/erp-$(date +%Y%m%d)

# 1. 数据库备份
docker exec erp-mysql mysqldump -uroot -p你的密码 industrial_erp > /vol1/docker/erp-system/db_backup_$(date +%Y%m%d).sql

# 2. 拉新代码
git pull origin main

# 3. 滚动重启 (数据卷不丢)
docker compose up -d --build

# 4. 如有 SQL 变更, 按文件名顺序导入
docker exec -i erp-mysql mysql -uroot -p你的密码 industrial_erp < sql/新文件.sql
```

## 十二、与群晖部署对比

| 维度 | 群晖 Container Manager | 飞牛 OS |
|---|---|---|
| 系统授权 | DSM 套件要"机型匹配", 部分机型无法装新版本 | **完全免费**, 无授权 |
| Docker 引擎 | 定制版, 部分镜像标签不兼容 (本项目遇过 `node:20-alpine` 解析失败) | 标准 Docker CE, 无兼容性问题 |
| Compose 支持 | 仅 V2, GUI 创建项目时偶尔有 bug | 原生 V2, 桌面图标点 3 下就行 |
| 文件系统 | btrfs (DSM 7+) | btrfs (默认) |
| 快照 | DSM 共享文件夹快照 | btrfs subvolume 快照 |
| 资源占用 | 较重 (DSM 套件多) | 较轻 (基础系统只 1G 内存) |
| 反向代理 | 控制面板 → 登录门户 | 设置 → 反向代理 |
| 证书 | 安全性 → 证书 | 设置 → 证书 |
| 移动 App | Synology Photos / Drive / DS File | fnOS App (持续完善中) |
| 适用人群 | 习惯群晖生态、不想折腾 | 想省钱、想装到老机器、想纯国产 |

**结论**: 本项目在飞牛上部署**比群晖更顺畅**, docker-compose.yml 直接用, 无需修改镜像标签.

## 十三、常见问题

### 13.1 fnOS.local 打不开
- 用 `http://NAS_IP` 直接访问 (路由器看 DHCP 分配)
- 或安装 fnConnect (官网下载) 自动发现

### 13.2 docker compose 命令找不到
- fnOS 默认 SSH 后是普通用户, 需要 `sudo -i` 进 root
- 验证: `docker compose version` 应输出 v2.x

### 13.3 端口 80 被占
- 飞牛桌面自身占用 80 / 443 (Web UI)
- 解决: `docker-compose.yml` 里 pc-web 改成 `"8880:80"`, 然后用 `http://NAS_IP:8880` 访问
- 或: 反向代理用 8080 转 80, pc-web 还是 80 (反代会接管 80)

### 13.4 MySQL OOM 被杀
8G 内存机型容易出现, 编辑 `docker-compose.yml`:
```yaml
command:
  - --innodb-buffer-pool-size=384M    # 原 512M 调到 384M
```
后端堆同步调小:
```yaml
JAVA_OPTS: "-Xms256m -Xmx512m -XX:MaxMetaspaceSize=160m ..."
```

### 13.5 想清空所有数据重来
```bash
cd /vol1/docker/erp-system
docker compose down -v        # -v 删数据卷, 慎用!
rm -rf data/
docker compose up -d --build
```

### 13.6 飞牛上 docker hub 拉镜像慢
- **设置** → **Docker** → **镜像加速** → 填阿里云/腾讯云镜像地址
- 或 SSH 修改 `/etc/docker/daemon.json` 加 `registry-mirrors`
