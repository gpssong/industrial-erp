# 部署方案与启动说明

## 一、环境要求

| 组件 | 版本 | 说明 |
|---|---|---|
| JDK | 17+ | 后端运行时 |
| Maven | 3.8+ | 构建 |
| MySQL | 8.0+ | 字符集 utf8mb4 |
| Redis | 6.0+ | 缓存 / 分布式锁 |
| Node.js | 18+ | 前端构建 |
| HBuilderX | 3.8+ | 移动端 (可选) |
| Electron | 29+ | 桌面端 (可选) |

## 二、数据库初始化

```bash
# 登录 MySQL
mysql -uroot -p

# 创建数据库
CREATE DATABASE industrial_erp DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
exit

# 导入 SQL (按顺序)
cd erp-system/sql
mysql -uroot -p industrial_erp < 01_schema_system.sql
mysql -uroot -p industrial_erp < 02_schema_base.sql
mysql -uroot -p industrial_erp < 03_schema_purchase.sql
mysql -uroot -p industrial_erp < 04_schema_sales.sql
mysql -uroot -p industrial_erp < 05_schema_inventory.sql
mysql -uroot -p industrial_erp < 06_schema_production.sql
mysql -uroot -p industrial_erp < 07_schema_outsource_finance.sql
mysql -uroot -p industrial_erp < 08_schema_misc.sql
mysql -uroot -p industrial_erp < 09_seed_data.sql

# 或一条命令
cat 0*.sql | mysql -uroot -p industrial_erp
```

## 三、后端启动

```bash
cd erp-system/backend

# 修改 application.yml 中的 datasource / redis 配置
# url: jdbc:mysql://你的IP:3306/industrial_erp?...
# username: root
# password: 你的密码

# 编译
mvn clean package -DskipTests

# 运行
java -jar target/industrial-erp.jar --spring.profiles.active=prod

# 或开发模式
mvn spring-boot:run
```

启动后:
- API: http://localhost:8080/api
- 接口文档: http://localhost:8080/api/doc.html

## 四、PC Web 启动

```bash
cd erp-system/pc-web

# 安装依赖
npm install   # 或 pnpm install / yarn

# 开发模式
npm run dev   # http://localhost:5173

# 生产构建
npm run build  # 输出到 dist/
```

将 `dist/` 部署到 Nginx:
```nginx
server {
    listen 80;
    server_name erp.example.com;
    root /opt/industrial-erp/pc-web;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8080/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        client_max_body_size 100M;
    }
}
```

## 五、Electron 桌面端打包

```bash
# 1. 先构建 PC Web
cd erp-system/pc-web && npm run build

# 2. 安装 Electron 依赖
cd ../electron && npm install

# 3. 开发模式 (需 Vite dev server 启动)
VITE_DEV_SERVER_URL=http://localhost:5173 npm run dev

# 4. 打包
npm run build:win    # Windows 安装包 (.exe)
npm run build:mac    # macOS DMG
npm run build:linux  # Linux (AppImage)

# 输出在 dist/ 目录
```

## 七、完整部署 (生产)

```
┌────────────────────────────────────────────┐
│  Nginx (80/443)                             │
│  ├─ /          -> pc-web/dist (静态)        │
│  └─ /api/      -> 反代到 8080                │
└────────────────┬───────────────────────────┘
                 │
        ┌────────┴────────┐
        ▼                 ▼
   Backend (8080)    Redis (6379)
        │
        ▼
   MySQL (3306)
```

启动脚本 `start.sh`:
```bash
#!/bin/bash
# 启动 Redis
redis-server --daemonize yes

# 启动 MySQL (容器化)
docker run -d --name mysql -e MYSQL_ROOT_PASSWORD=xxx -p 3306:3306 mysql:8.0

# 启动后端
nohup java -jar -Xms512m -Xmx2g industrial-erp.jar > app.log 2>&1 &

# 启动 Nginx
nginx -g 'daemon off;'
```

## 九、监控与运维

- **日志**: `/var/log/industrial-erp/` (建议挂载)
- **慢SQL**: MySQL slow_query_log
- **JVM**: 开启 JMX, 接入 Prometheus
- **APM**: SkyWalking (可选)


## 十、版本升级

```bash
# 1. 备份
mysqldump industrial_erp > backup_$(date +%Y%m%d).sql

# 2. 拉取新代码
git pull origin main

# 3. 执行新 SQL (如果)
mysql industrial_erp < sql/migrations/v1.1.0.sql

# 4. 重新构建
cd backend && mvn clean package -DskipTests
cd ../pc-web && npm run build

# 5. 重启服务
systemctl restart industrial-erp
```

## 十一、常见问题

**Q1. 启动后无法连接数据库?**
A: 检查 `application.yml` 中 `spring.datasource.url` 用户名/密码, 以及 MySQL 是否启动。

**Q2. Redis 报错?**
A: 启动 Redis, 确认 `spring.data.redis.host` 正确。

**Q3. 接口 401?**
A: Token 过期, 重新登录; 或 Sa-Token 配置的 `exclude` 路径不正确。

**Q4. 库存出库失败, 提示库存不足?**
A: 业务上禁止负库存; 请先入库或调整单据数量。

**Q5. 打印不显示?**
A: 后端 `PrintService` 需能访问, 模板路径在 `resources/templates/print/`。

**Q6. App 无法连接?**
A: 修改 `app/api/index.js` 的 `BASE` 地址为生产后端。
